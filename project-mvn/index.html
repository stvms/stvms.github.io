Mixture of Experts - A MultiView Node Infinite Canvas

A workspace for ideas and concepts using machine learning to support your creations with a single global element: The MultiView Node.

The MultiView Node has two primary views and a default cover view to hide/collapse the content. The text view, designed for prompting and the image view for generated results. Furthermore the image view allows image prompting and image understanding, either with a generated result or with your own images for greater flexiblity. With that being said, a source and target handle on its corresponding sides allows to build a network of MultiView Nodes together.

Infinite Canvas with MultiView Nodes, also the custom Context Menu next to it.

Making this style of work possible are 4 types of experts that handle different kinds of user input in the background. These include models for image generation, image-based prompting, visual understanding, and a unimodal language model dedicated to Text-to-Text processing.
Those 4 types are the core workflow strategy and were chosen to create a closed loop of operations. Selectable as modes in the parameter section of a node.

To demonstrate a cycle here is an example: Start with a Text-to-Image workflow to create an image from a prompt. Afterwards use the Image-to-Text mode to describe it, then refine that description with the Text-to-Text mode. Finally, reuse the improved text in another Text-to-Image process using a new connected MultiView Node - and continue exploring.

Basically allowing chains with the goal to iterate or branch out of previous concepts. This approach also allows to 'chat' (Text-to-Text chained x times etc.) non-linearly in comparison to current interaction with large language models. Creating a communication tree.

Features Showcase

Below is a collection of features that are packed into this tool. Aimed to improve the overall workflow on the canvas.

MultiView Node runtime animation - For visual feedback indicating that the prompt is being processed. Additionally the corresponding node handle will pulse to show the target direction. Either incoming or outgoing - decided by the user and on the content of the connected nodes.

Layout Management - If the scene gets cluttered by multiple different nodes, just select them for layout handling. Either manually or via 'Select All'. Afterwards using the context menu selecting the 'Pack & Optimize' option will spread the nodes evenly across the canvas.

Tidy up your workspace and continue your flow.

Supports light and dark mode for all nodes, control elements and the canvas itself. Changeable via the context menu.

Behind the Scenes

Custom ComfyUI 4x1 Workflow

Here is an overview of the backend workflow engine that gets called, when the node is executed with one of its available modes. This is possible with the help of the ComfyUI developer API and also allows to modify the JSON to correctly switch the model context, before sending the request.

The custom ComfyUI 4x1 workflow, consisting of 4 experts: Text-2-Image, Image-2-Image, Image-2-Text and Text-2-Text

Note: It is important to mention, that after each run the model and execution cache will be cleared - due to lower hardware specs (6GB VRAM), during development. While this increases the time a bit for each generation, it enables fitting different models into the VRAM, when switching and prevents any remaining resources of a previous run occupying the memory.
